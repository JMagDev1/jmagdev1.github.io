Finance Web App

Here you can see an overview of the whole page of my finance web app. There are several applicaions for this app: a tracker to tackk money in and money out, a savings section: set a target add amount in and out, and a monthly bill tracker. Idealy I would have liked to save the information in this to a database then to request all the informatino back so that when the user closes it the information is not lost. However due to my knowledge at the time of undertaking this project I didnt know how to add any back end code to an application. Whenever I learn more on the safety required when working with transactions I plan to re visit this project and complete it properly.

When the user wants to add either an expense or income they can use the form at the bottom of the page. There is a dropdown menu to select weather it is money in or out and then textboxes to add the amount, category, where and the date of the exchange. This information is then added to the main table seen in at the top left. If the exchange is a purchase then it gets added to a secondary table at the top right of the screen. This is to keep track of all expenses only.

This section of the web app is designed for savings. Here the user can enter a savings target, add money into thier savings and also take money out of their savings. If they have met thier target the text will change to green to let the user know they have met the target.
